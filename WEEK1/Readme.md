# 🔐 Week 1: Identity and Access Management (IAM) Setup

## Overview
The first week focused on securing the AWS environment. Following the **Principle of Least Privilege (PoLP)**, I moved away from using the Root account for daily tasks and established a secure IAM infrastructure.

## Key Activities
* **Root Security:** Enabled Multi-Factor Authentication (MFA) on the Root account to prevent unauthorized access.
* **IAM User Creation:** Created an administrative IAM user (`admin_user`) for daily project management.
* **Groups and Permissions:** Organized users into functional groups with specific policies attached to ensure secure access control.
